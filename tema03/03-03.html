Resolución de nombres

3.3. Resolución de nombres

Cuando usamos Internet o cualquier red, hay dos formas de identificar un equipo (host):

  • Por su nombre: Es lo que usamos normalmente en el nivel de aplicación. Por ejemplo, "www.wikipedia.org" es fácil de recordar y escribir.
  • Por su dirección IP: Es lo que realmente necesitan los niveles inferiores (transporte y red) para establecer la comunicación. Por ejemplo, "91.198.174.192".

Imagina que quieres visitar http://www.wikipedia.org usando tu navegador web. Tú escribes "www.wikipedia.org" porque es fácil de recordar, pero tu computadora necesita conocer la dirección IP real de ese servidor para poder conectarse con él. Aquí es donde entra en juego la resolución de nombres: el proceso que traduce nombres legibles (como www.wikipedia.org) en direcciones IP numéricas.

Los sistemas operativos incluyen un servicio especial de resolución de nombres que se encarga de esta traducción. Este servicio utiliza dos mecanismos diferentes que veremos a continuación:

3.3.1. Resolución de nombres local

Este mecanismo funciona mediante archivos locales almacenados en tu propia computadora. Estos archivos contienen una lista de asociaciones entre nombres y direcciones IP, como si fuera una pequeña agenda de contactos.

Los archivos más comunes son:

  • En Linux y macOS: /etc/hosts
  • En Windows: C:\Windows\System32\drivers\etc\hosts

Veamos un ejemplo del contenido de este archivo:

127.0.0.1       localhost

192.168.5.17    kabuto
192.168.5.19    portatil_de_luis
192.168.5.31    leonardo

# Las siguientes líneas son para hosts con capacidad IPv6
::1     ip6-localhost ip6-loopback
fe00::0 ip6-localnet
ff00::0 ip6-mcastprefix
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters

¿Qué significa este archivo?

  • La primera línea indica que el nombre "localhost" corresponde a la dirección IP 127.0.0.1, que siempre apunta a tu propia máquina.
  • Las líneas siguientes asocian nombres personalizados (como "kabuto" o "portatil_de_luis") con direcciones IP de tu red local.
  • Las líneas que comienzan con # son comentarios explicativos.
  • Las últimas líneas son para direcciones IPv6, la versión más moderna del protocolo de Internet.

Gracias a este archivo, puedes acceder a recursos usando nombres en lugar de números. Por ejemplo:

  • Escribir http://localhost en lugar de http://127.0.0.1 para acceder a un servidor web en tu propia máquina.
  • Escribir http://leonardo en lugar de http://192.168.5.31 para acceder a otro equipo en tu red local.

3.3.2. El protocolo DNS

DNS (Sistema de Nombres de Dominio, por sus siglas en inglés Domain Name System) es un protocolo que permite traducir nombres de dominio en direcciones IP, pero a diferencia del método anterior, no se limita a tu computadora: funciona consultando servidores especializados en Internet.

Este es el mecanismo que usamos para acceder a sitios web como www.wikipedia.org. Cuando escribes este nombre en tu navegador, tu computadora consulta automáticamente a un servidor DNS para obtener la dirección IP correspondiente.

Ejemplo práctico: Podemos consultar manualmente las direcciones IP de un sitio web usando el comando nslookup. En este ejemplo, consultamos al servidor DNS público de Google (8.8.4.4) para obtener las direcciones IPv4 e IPv6 de www.wikipedia.org:

$ nslookup www.wikipedia.org 8.8.4.4
Server:
        8.8.4.4
Address:        8.8.4.4#53

Non-authoritative answer:
www.wikipedia.org       canonical name = dyna.wikimedia.org.
Name:   dyna.wikimedia.org
Address: 91.198.174.192
Name:   dyna.wikimedia.org
Address: 2620:0:862:ed1a::1

¿Qué nos muestra este resultado?

  • Server: El servidor DNS que respondió a nuestra consulta (8.8.4.4 en el puerto 53, que es el puerto estándar de DNS).
  • Canonical name: El nombre real del servidor (dyna.wikimedia.org), ya que www.wikipedia.org es un alias que apunta a él.
  • Address: Las direcciones IP encontradas. En este caso, obtuvimos una dirección IPv4 (91.198.174.192) y una dirección IPv6 (2620:0:862:ed1a::1).

Configuración de DNS en tu equipo:

Para poder navegar eficazmente por Internet, es fundamental que tu computadora esté correctamente configurada como cliente DNS. Esta configuración consiste en especificar la dirección IP de uno o más servidores DNS que tu equipo consultará cuando necesite traducir nombres a direcciones IP.

En 2025, algunos de los servidores DNS públicos más utilizados son:

  • Google Public DNS: 8.8.8.8 y 8.8.4.4
  • Cloudflare DNS: 1.1.1.1 y 1.0.0.1 (destacado por su velocidad y privacidad)
  • Quad9: 9.9.9.9 (enfocado en seguridad, bloqueando sitios maliciosos)
  • OpenDNS: 208.67.222.222 y 208.67.220.220

La mayoría de los usuarios obtienen esta configuración automáticamente de su proveedor de Internet (ISP) o de su router doméstico, aunque también es posible configurar manualmente servidores DNS específicos para mejorar la velocidad, privacidad o seguridad de tu navegación.
