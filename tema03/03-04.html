3.4. Clases de Java para comunicaciones en red

Java incluye un paquete llamado java.net que contiene herramientas (clases) especialmente diseñadas para que los programas puedan comunicarse a través de una red. Estas herramientas funcionan con los protocolos estándares de Internet, que pertenecen a la familia TCP/IP (los mismos que utiliza tu navegador web o tu aplicación de mensajería).

Este paquete se puede organizar en dos grandes grupos de clases:

  1. Clases de bajo nivel: Son las clases fundamentales que representan los elementos básicos de las redes, como tarjetas de red, direcciones IP y conexiones. Estas clases trabajan a nivel de los protocolos más cercanos al hardware (niveles de enlace, red y transporte). El cuadro 3.4 muestra estas clases, que son las que estudiaremos en este capítulo.
  2. Clases de alto nivel: Están relacionadas con protocolos más especializados que funcionan a nivel de aplicación (como HTTP para páginas web o FTP para transferencia de archivos). Algunas de estas clases se estudiarán en el siguiente capítulo.

En este capítulo haremos primero un repaso breve de las clases que manejan interfaces de red y direcciones IP. Después nos concentraremos en lo más importante: las clases para crear sockets (puntos de conexión) tanto de UDP como de TCP. Estas clases son el núcleo de este capítulo.

Con estas herramientas desarrollaremos programas que actúan como servidores (los que esperan conexiones) y como clientes (los que inician las conexiones).

Estos programas se comunicarán usando los dos principales protocolos de transporte: UDP y TCP. Aunque serán ejemplos sencillos, estaremos creando nuestros propios protocolos de nivel de aplicación básicos.

CUADRO 3.4

Clases de bajo nivel del paquete java.net

Interfaces de red NetworkInterface: Representa una interfaz de red física o virtual de tu ordenador (como la tarjeta WiFi, la tarjeta Ethernet, o conexiones virtuales).
Direcciones IP InetAddress: Representa cualquier dirección IP (la "matrícula" que identifica a un dispositivo en la red).
Inet4Address: Representa específicamente una dirección IPv4 (el formato tradicional, como 192.168.1.1).
Inet6Address: Representa específicamente una dirección IPv6 (el formato moderno y más extenso, como 2001:0db8:85a3::8a2e:0370:7334).
InterfaceAddress: Combina una dirección IP con información adicional sobre la red a la que pertenece esa interfaz (como la máscara de subred).
Sockets Socket: Es un socket TCP del lado del cliente. Se utiliza cuando quieres conectarte a un servidor mediante el protocolo TCP (protocolo confiable que garantiza que los datos lleguen en orden).
ServerSocket: Es un socket TCP del lado del servidor. Se queda "escuchando" y acepta conexiones entrantes de clientes que usan TCP.
DatagramSocket: Es un socket para el protocolo UDP. Se usa tanto para enviar como para recibir datagramas (paquetes de datos independientes) mediante UDP (protocolo más rápido pero sin garantías de entrega).

Nota sobre versiones actuales (2025): A partir de Java 19 (2022) se introdujeron mejoras en las API de red, incluyendo Virtual Threads que permiten manejar múltiples conexiones de forma más eficiente. Las clases mencionadas siguen siendo las fundamentales, aunque ahora se recomienda usar las APIs modernas de NIO.2 (java.nio) para aplicaciones que requieran alto rendimiento con muchas conexiones simultáneas.
